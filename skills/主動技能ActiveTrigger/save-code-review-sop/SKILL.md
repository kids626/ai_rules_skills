# Save Code Review SOP（程式碼意義解釋存檔）

## 目標

1. **封存解釋**：把**截至目前本對話**中，針對「某段／某檔程式」的說明（含逐行、邏輯、隱含假設）整理成一篇可搜尋的筆記。
2. **可回溯**：讀者不必翻聊天紀錄，也能知道**在講哪段程式**、**重點結論**、**相關檔案與符號**。
3. **檔名可找**：檔名含**日期＋主題／檔案／符號**線索，方便依檔案樹或關鍵字搜尋。

## 何時啟用

- **觸發詞 `save-code-review-sop`**：使用者訊息在 **trim 後、不分大小寫** 等於 `save-code-review-sop` 時（整則訊息僅此觸發詞），**立即**依「作業順序」執行：整理**截至目前對話**中的程式碼解釋，並寫入 `md文件檔/local_notes/`（勿只回覆確認而不產檔）。
- 使用者說「把剛剛那段解釋存起來」「存成筆記」「寫入 local_notes」且語意為**程式碼講解**而非整條流程／API 鏈時（後者請用 save-template-sop／savesop）。

## 作業順序（對代理）

1. **對齊主題**：用 1～2 句寫明「這篇筆記在解釋什麼」（例如：`UserService::login` 的驗證順序、某 React 元件的 effect 依賴）。
2. **還原脈絡**：從對話抽出：
   - **被解釋的程式**（檔案路徑、約略行號或符號名；若對話未寫明，從上下文或 repo 搜尋補上，不確處標「待確認」）。
   - **使用者的原始問題**（一句）。
   - **AI 給出的結論**（條列，避免只貼聊天口吻）。
3. **結構化內容**：依下方「輸出範本」填寫；若工作區規則要求可點行號，關鍵處附**精簡程式碼引用**（路徑＋行號範圍）。
4. **檔名規則（方便查找）**：寫入 `md文件檔/local_notes/`，檔名必須同時滿足：
   - 前綴日期：`YYYY-MM-DD-`（與使用者環境之「今天」一致；若未知則用對話日期或以 repo 內慣例為準）。
   - 主題段：kebab-case，**優先順序**為  
     `code-review-<相對路徑簡寫>-<符號或區塊簡稱>`  
     路徑簡寫規則：將 `src/foo/BarBaz.php` 轉成 `src-foo-BarBaz`（只保留最後一層檔名時可接受，但若重名風險高則多保留一層目錄）。  
     符號簡稱：如 `handleSubmit`、`useEffect-deps`、區塊語意 `validation-block`。  
     全檔名範例：`2026-05-13-code-review-src-services-Auth-login-flow.md`  
   - 若同一對話涉及多檔：以**使用者最關注的一檔**為主檔名，其餘檔在內文「相關檔案」列出；或使用者指定檔名時以使用者為準（仍應落在 `local_notes` 下）。
5. **落檔與回報**：目錄不存在則先建立 `md文件檔/local_notes`。以 UTF-8 寫入。完成後在對話中回報**實際儲存的完整相對路徑**（自 repo 根起算）。

## 落檔位置（本專案約定）

- **根路徑**（相對於 repo 根目錄）：`md文件檔/local_notes/`
- **用途**：程式碼意義、逐行說明、邏輯與邊界案例的複習筆記。
- **編碼**：UTF-8。

## 輸出範本（產出後寫入檔案）

```markdown
# 程式碼解釋筆記：<簡短標題>

## 1. 元資料
- 筆記日期：YYYY-MM-DD
- 對應問題（使用者原問）：…
- 主要程式位置：`path/to/file`（符號或行號範圍：…）

## 2. 一句總結
（這段程式在整體裡扮演什麼角色）

## 3. 分段說明（依對話實際內容調整）
### 3.1 在做什麼
### 3.2 資料怎麼流／狀態怎麼變（若有）
### 3.3 分支與邊界案例（若有）
### 3.4 與其他模組的關係（若有）

## 4. 關鍵程式碼引用（精簡）
（可點行號的引用區塊；只保留理解所需的行）

## 5. 名詞與假設（可選）
| 名詞／假設 | 說明 |
|------------|------|
| | |

## 6. 仍待確認／後續可讀
- …
```

## 注意

- 本 skill 專注**「程式在說什麼、為何這樣寫」**；若使用者要的是**整頁渲染鏈或 API 全鏈**，請改走 `save-template-sop`／`savesop`。
- 摘要以**未參與對話的人讀得懂**為準；避免只貼長篇聊天而無標題與結構。
- 若對話中沒有任何程式相關解釋，仍應產檔但於「仍待確認」說明「無可封存內容」，並建議使用者先完成提問再觸發。
