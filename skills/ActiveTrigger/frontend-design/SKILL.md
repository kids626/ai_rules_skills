此 skill 用於引導你打造風格鮮明、可上線使用的前端介面，避免落入通用且乏味的「AI 生成感」美學。請實作可運作的真實程式碼，並在視覺細節與創意選擇上保持高度講究。

使用者會提供前端需求：可能是元件、頁面、應用程式或介面。需求中也可能包含目標、受眾或技術限制等背景資訊。

## 設計思考

在寫程式前，先理解情境，並承諾採取一個**明確且大膽**的美學方向：
- **Purpose（目的）**：這個介面要解決什麼問題？誰會使用它？
- **Tone（調性）**：選擇一個鮮明極端，如：極簡冷冽、極繁混亂、復古未來、自然有機、精品精緻、玩具感、雜誌編輯感、粗獷野性、裝飾藝術幾何、柔和粉彩、工業機能等。你可以借鑑這些風格，但必須設計出真正符合本案方向的版本。
- **Constraints（限制）**：技術需求（框架、效能、無障礙等）。
- **Differentiation（差異化）**：什麼會讓它令人難忘？使用者最可能記住的那一點是什麼？

**CRITICAL（關鍵）**：選定清楚的概念方向，並精準執行。無論是大膽的極繁或克制的極簡都可行，重點在於「有意識的設計決策」，而非單純追求視覺強度。

接著實作可運作的程式碼（HTML/CSS/JS、React、Vue 等），並確保：
- 可投入正式環境且功能完整
- 視覺上有衝擊力並具記憶點
- 風格一致且美學立場明確
- 細節講究、打磨到位

## 前端美學準則

請重點關注：
- **Typography（字體）**：選擇有美感、獨特且有性格的字體。避免過度通用的字體（如 Arial、Inter）；改用能提升整體氣質的字體組合。可用一款具辨識度的展示字搭配一款精緻的內文字。
- **Color & Theme（色彩與主題）**：堅持統一且明確的美學。用 CSS 變數保持一致性。主色鮮明、搭配銳利重點色，通常比平均分配、保守的配色更有力量。
- **Motion（動態）**：善用動畫與微互動。純 HTML 場景優先考慮 CSS 動畫；React 可用 Motion 類函式庫。聚焦高影響時刻：一個設計良好的進場節奏（含 stagger 與 animation-delay）常比零散微動畫更有驚喜。結合滾動觸發與 hover 狀態提升互動感。
- **Spatial Composition（空間構圖）**：嘗試非預期版面、非對稱、重疊、斜向流動、突破網格的元素，以及大面積留白或可控高密度編排。
- **Backgrounds & Visual Details（背景與視覺細節）**：不要只用單色背景，應建立氛圍與層次。加入符合主題的紋理與效果，例如漸層網格、噪點質感、幾何圖樣、半透明分層、戲劇化陰影、裝飾邊框、自訂游標與顆粒覆蓋等。

**絕對避免**通用化的 AI 美學：被濫用的字體家族（如 Inter、Roboto、Arial、系統字體）、陳腔濫調配色（特別是白底紫漸層）、可預測的版型與元件套路，以及缺乏情境特色的模板式設計。

請以創意詮釋需求，做出「符合情境、且出乎意料」的設計決策。每次產出都應該不同：在明暗主題、字體選擇與整體美學上持續變化。**不要**在多次生成中反覆收斂到同一組常見選項（例如 Space Grotesk）。

**IMPORTANT（重要）**：實作複雜度要與美學願景匹配。極繁風格需要更完整的動畫與效果程式；極簡或精緻風格則需要克制、精準，以及對留白、字體、細節的高度掌握。真正的優雅來自對方向的高品質執行。

請記得：你具備非常高的創意上限。不要保守，勇於跳脫慣性，並全力投入一個鮮明設計願景，做出真正令人驚豔的作品。
