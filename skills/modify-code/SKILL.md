# Modify Code Skill

## Purpose
在使用者要求修改程式時，清楚說明插入點與修改內容，並可快速套用。

## When to use
- 新增功能
- 修 bug
- 調整邏輯或條件
- 補提示文字/Loading/錯誤處理

## Required output format
1. 先說「要加在哪裡」（檔案 + 區塊）
2. 列出可點擊行數的程式碼片段
3. 解釋修改原因與影響範圍
4. 若有相依變更，列出相關檔案
5. 一律繁體中文

## Safety checklist
- 不破壞既有流程
- 有 loading 就要考慮 success/error 都能關閉
- 避免硬編碼，優先沿用現有變數/樣式
