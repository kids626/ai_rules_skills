# Logic Trace Skill

## Purpose
在使用者詢問「程式邏輯」時，快速給出流程、入口、關鍵條件與對應程式位置。

## When to use
- 問「這頁怎麼跑」
- 問「搜尋如何觸發」
- 問「資料從哪來、怎麼送出」
- 問「為什麼會進到這個分支」

## Required output format
1. 先給流程摘要（2~5點）
2. 列出每個步驟的程式碼位置（可點擊行數）
3. 說明關鍵條件與資料流向
4. 指出可能的例外或邊界情況
5. 一律繁體中文

## Trace template
- 入口：UI事件 / API呼叫
- 處理：參數整理 / 條件判斷
- 輸出：畫面更新 / 資料寫入 / 錯誤處理
