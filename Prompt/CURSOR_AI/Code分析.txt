﻿AI 提問模板（程式碼詳解 / 修改 / 除錯）

使用方式：把「你要分析的檔案/程式碼」貼上後，再選下面一種問題模板。可以混搭。

========================================
一、完整解析整支程式碼（深度）
========================================
請完整解析這份檔案，請用「分段 + 逐行」方式說明，包含：
1. 每一段程式碼的功能與目的
2. 函式之間的呼叫關係與資料流
3. 程式從入口到結束的執行順序
4. 外部依賴的來源（API、store、props、路由參數等）
5. 隱含邏輯或容易忽略的假設
最後請補上：
- 可優化之處
- 若要重構，你的建議方向

========================================
二、逐行翻譯 + 通俗註解（超細）
========================================
請逐行解釋這段程式碼，每一行都要說明：
- 這行在做什麼
- 對應到整體流程中的哪個階段
- 如果刪掉會造成什麼影響
另外請補充：
- 哪些是語法糖
- 哪些是框架特有語法
- 哪些可能會出錯或有副作用

========================================
三、流程圖 + 執行路徑
========================================
請根據程式碼生成流程圖（文字版即可），並在每個節點說明：
- 進入條件
- 執行內容
- 可能的例外
如果資訊不足，請列出需要我補充的檔案或函式名稱。

========================================
四、定位問題 / 分析 Bug
========================================
請幫我從以下角度檢查程式碼：
- 可能造成 bug 的邏輯錯誤
- 例外處理不足的地方
- 資料流程不一致的風險
- 效能瓶頸
請列出風險點，並附上改善建議。

========================================
五、功能新增（欄位 / 搜尋 / 列表）
========================================
我要新增一個欄位或搜尋條件，請用以下格式回答：
1. 需要修改哪些檔案（含路徑）
2. 每個檔案要新增或修改哪一段
3. 標出插入位置（行數或相對區塊）
4. 提供修改後的範例程式碼
5. 若需改後端 API 或資料表，也請說明

========================================
六、後端 API / 資料表 / SQL 查詢
========================================
請幫我查後端 API 使用的資料表與 SQL 篩選條件，回答包含：
- Controller / Service / Repository 的位置
- 主要查詢的 SQL（或查詢條件）
- 使用的 DB 與資料表名稱

========================================
七、專案級理解（多檔案）
========================================
我會提供多個檔案。請在分析時持續整理：
1. 路由與控制器的對應
2. 函式呼叫鏈
3. 服務與 Repository 的職責拆分
4. Model 與資料流向
請在我逐步貼文件時持續更新。

========================================
八、我要你先統整再結論
========================================
請先摘要與統整，不要急著生成結論。
我會再告訴你要深入的部分。
